# 贡献指南

欢迎参与本项目的改进。建议遵循以下步骤：

## 如何提交 issue
- 在仓库的 Issues 列表中新建 issue，说明问题或建议，并尽可能提供复现步骤、预期行为和截图。

## 如何贡献代码
1. Fork 本仓库到你自己的账户。
2. 从 main 分支创建一个新分支，命名为 `feature/<简短描述>` 或 `fix/<简短描述>`。
3. 在该分支上进行修改或新增文档。每次提交请使用清晰的提交信息，建议包含变更内容和背景。
4. 完成修改后推送到你的远程仓库。
5. 在原仓库发起 Pull Request，描述变更的动机、解决的问题和相关 issue 编号。请勾选检查清单确认已遵循本指南。
6. 等待维护者的代码审查。维护者可能会提出修改意见，请及时回应和更新。
7. 审核通过后会合并到 main 分支。感谢你的贡献！

## 编码与文档风格
- Markdown 文档请使用简洁的语言，注意标题层级和列表格式。
- 对于代码片段，请保持一致的缩进和格式。
- 提交信息建议使用中文，并以动词开头，例如“修复拼写错误”、“新增内容制作 SOP 文档”。

## 行为守则
请保持尊重与包容，遵守 GitHub 社区准则。不接受带有人身攻击、歧视或冒犯性内容的贡献。
